palmwtc.notebooks_runner#

palmwtc.notebooks_runner — Papermill-mode pipeline orchestrator.

Ported from flux_chamber/scripts/run_notebooks.py with these changes:

  • Notebook directory is now resolved via DataPaths.extras['notebooks_dir'] rather than walking up from the script location.

  • Output directory mirrors that resolution.

  • Spine/parallel split preserved exactly.

  • Thread-saturation env vars set as before.

  • BATCH_MODE=1 set by default so widget cells gracefully skip.

The library-mode equivalent (no papermill, faster, headless) lives in palmwtc.pipeline. Use this module when you want the full executed HTML reports as documentation of a run.

Run all notebooks in notebooks_dir, sorted by numeric prefix.

See flux_chamber/scripts/run_notebooks.py for the original CLI shape. The behaviour here is byte-equivalent: spine notebooks (010/020/030/040) run sequentially first, then non-spine notebooks run with parallel workers (default 1 = sequential).
